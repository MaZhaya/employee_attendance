## 1. 这个系统是干什么的？

这是一个 **员工考勤管理系统**（Web 应用），可以完成以下功能：

| 功能 | 说明 |
|------|------|
| 🔐 登录/注册 | 分两种角色：`admin`（管理员）和 `employee`（普通员工）|
| 👨‍💼 员工管理 | 管理员可以增删改查员工信息 |
| 🏢 部门管理 | 管理员可以管理公司部门 |
| ⏰ 打卡签到 | 员工上班打卡、下班签退，自动判断是否迟到 |
| 📋 请假申请 | 员工提交请假，管理员审批/拒绝 |
| 📊 考勤查看 | 员工看自己的考勤记录，管理员看所有人的 |

登录后根据角色跳转不同页面：

```
登录 → 是管理员吗？
         ├── 是 → /admin/index.jsp    （管理后台）
         └── 否 → /employee/index.jsp  （员工首页）
```

## 4. 核心概念：三层架构

这个项目采用了 Java Web 中最经典的 **三层架构**（其实是四层）：

```
┌──────────────────────────────────────────────┐
│  浏览器 (用户操作)                              │
└──────────────┬───────────────────────────────┘
               │ HTTP 请求
               ▼
┌──────────────────────────────────────────────┐
│  Servlet 层 (控制器)                          │  ← 第 ① 层
│  接收请求、调用 Service、决定跳转哪个页面        │
│  LoginServlet / EmpServlet / LeaveServlet...  │
└──────────────┬───────────────────────────────┘
               │ 调用
               ▼
┌──────────────────────────────────────────────┐
│  Service 层 (业务逻辑)                         │  ← 第 ② 层
│  处理业务规则（判断迟到、审批逻辑...）            │
│  EmployeeServiceImpl / AttendanceServiceImpl  │
└──────────────┬───────────────────────────────┘
               │ 调用
               ▼
┌──────────────────────────────────────────────┐
│  Mapper 层 (数据访问)                          │  ← 第 ③ 层
│  执行 SQL 语句，读写数据库                      │
│  EmployeeMapper.xml (+ 接口)                  │
└──────────────┬───────────────────────────────┘
               │ JDBC
               ▼
┌──────────────────────────────────────────────┐
│  MySQL 数据库                                 │  ← 第 ④ 层
│  employee / department / attendance ...       │
└──────────────────────────────────────────────┘
```

**为什么要分层？** 和餐厅类比：

| 层 | 餐厅角色 | 职责 |
|----|----------|------|
| Servlet | 服务员 | 接待顾客，传菜单 |
| Service | 厨师 | 按菜谱做菜（业务逻辑）|
| Mapper | 食材管理员 | 去仓库取食材（数据库操作）|
| Database | 仓库 | 存储食材（数据） |

分层的好处：修改数据库不影响服务员，修改菜谱不影响仓库管理。

## 5. 第一层：实体层 Entity（数据模型）

实体类的特点是：**一个类对应数据库中的一张表，一个属性对应表中的一个字段**。

### 5.1 Employee.java — 员工实体

```java
public class Employee {
    private Integer id;        // 员工ID（主键）
    private String empNo;      // 员工编号（如 "EMP001"）
    private String empName;    // 姓名
    private String gender;     // 性别
    private String phone;      // 电话
    private Integer deptId;    // 所属部门ID（外键，关联部门表）
    private Date entryDate;    // 入职日期
    private Integer status;    // 状态（1=在职，0=离职）

    // 每个属性都有对应的 getter 和 setter 方法
    // getter = 读取属性值
    // setter = 设置属性值
}
```

> 💡 **通俗理解**：Entity 就像一个"数据容器"，从数据库中取出一行数据，就装进这样一个对象里，方便 Java 代码操作。

### 5.2 Department.java — 部门实体

```java
public class Department {
    private Integer id;        // 部门ID
    private String deptName;   // 部门名称（如 "技术部"）
    private String deptDesc;   // 部门描述
}
```

### 5.3 Attendance.java — 考勤实体

```java
public class Attendance {
    private Integer id;        // 记录ID
    private Integer empId;     // 员工ID
    private Date checkDate;    // 考勤日期
    private Time checkIn;      // 上班打卡时间
    private Time checkOut;     // 下班签退时间
    private String status;     // 状态（"正常" 或 "迟到"）
    private String remark;     // 备注

    // ⭐ 多对一关联：一条考勤记录对应一个员工
    private Employee employee;
}
```

> 💡 **多对一关联**：考勤表的 `employee` 属性可以装一个完整的 Employee 对象。这样查考勤时，可以同时拿到这条考勤是谁的。在 MyBatis 中使用 `<association>` 标签实现。

### 5.4 LeaveApply.java — 请假实体

```java
public class LeaveApply {
    private Integer id;          // 申请ID
    private Integer empId;       // 员工ID
    private String leaveType;    // 请假类型（事假/病假/年假）
    private Date startTime;      // 开始日期
    private Date endTime;        // 结束日期
    private String reason;       // 请假原因
    private String approveStatus; // 审批状态（待审批/已通过/已拒绝）
}
```

### 5.5 User.java — 用户实体（登录用）

```java
public class User {
    private Integer id;        // 用户ID
    private String username;   // 用户名
    private String password;   // 密码
    private String role;       // 角色（"admin" 或 "employee"）
    private Integer empId;     // 关联的员工ID
}
```

> ⚠️ **注意**：`User`（登录账号）和 `Employee`（员工信息）是分开的两张表。一个登录账号通过 `empId` 关联到一个员工记录。

## 6. 第二层：数据访问层 DAO/Mapper（操作数据库）

这一层负责和数据库交互。项目中使用了 **MyBatis** 框架。

### MyBatis 是什么？

MyBatis 是一个 **半自动的 ORM 框架**，它的核心思想是：

```
Java 接口方法  ←→  XML 中的 SQL 语句
```

你只需要写一个接口方法和对应的 SQL，MyBatis 帮你执行。

### 6.1 工作流程

```
① 定义 Mapper 接口（Java 方法签名）
         ↕ 绑定
② 编写 Mapper XML（SQL 语句）
         ↕ 执行
③ MyBatisUtil 获取 SqlSession，执行 SQL
```

### 6.2 接口 + XML 配对示例

**EmployeeMapper.java（接口）：**

```java
public interface EmployeeMapper {
    boolean addEmployee(Employee employee);     // 添加员工
    boolean deleteEmployee(@Param("id") Integer id);  // 删除员工
    boolean updateEmployee(Employee employee);  // 修改员工
    Employee getEmployeeById(@Param("id") Integer id); // 查询单个
    List<Employee> listAll();                   // 查询所有
}
```

**EmployeeMapper.xml（SQL 语句）：**

```xml
<mapper namespace="mapper.EmployeeMapper">  <!-- namespace 指向接口 -->

    <!-- 添加员工：#{xxx} 会被替换为 Employee 对象的属性值 -->
    <insert id="addEmployee">  <!-- id 对应接口方法名 -->
        INSERT INTO employee(emp_no, emp_name, gender, phone, dept_id, entry_date, status)
        VALUES(#{empNo}, #{empName}, #{gender}, #{phone}, #{deptId}, #{entryDate}, #{status})
    </insert>

    <!-- 删除员工 -->
    <delete id="deleteEmployee">
        DELETE FROM employee WHERE id=#{id}
    </delete>

    <!-- 查询所有员工 -->
    <select id="listAll" resultType="entity.Employee">
        SELECT * FROM employee
    </select>
</mapper>
```

> 💡 **关键对应关系**：
> - `namespace` → 接口的全路径
> - `id` → 接口中的方法名
> - `#{xxx}` → 方法参数或对象的属性名
> - `resultType` → 查询结果转换成什么类型的对象

### 6.3 考勤模块的高级用法 — 关联查询

AttendanceMapper.xml 中有一个 **关联查询**，一次 SQL 查出考勤 + 员工信息：

```xml
<!-- 定义结果映射 -->
<resultMap id="AttendanceEmployeeMap" type="entity.Attendance">
    <id column="id" property="id"/>
    <result column="emp_id" property="empId"/>
    <!-- ... 其他字段 ... -->

    <!-- ⭐ 多对一关联：把 e 表的字段映射到 employee 属性里 -->
    <association property="employee" javaType="entity.Employee">
        <id column="e_id" property="id"/>
        <result column="emp_name" property="empName"/>
        <!-- ... -->
    </association>
</resultMap>

<!-- 关联查询 SQL -->
<select id="listAllWithEmployee" resultMap="AttendanceEmployeeMap">
    SELECT
        a.*,
        e.id AS e_id,      -- 给别名，防止和 a.id 冲突
        e.emp_name,
        e.gender,
        e.phone
    FROM attendance a
    LEFT JOIN employee e ON a.emp_id = e.id
</select>
```

> 💡 **通俗理解**：`LEFT JOIN` 把考勤表和员工表"拼"在一起查，一次拿到全部信息。`resultMap` 告诉 MyBatis 怎么把查出来的列映射到 Java 对象的属性上。

### 6.4 MyBatisUtil.java — 数据库连接工具

```java
public class MyBatisUtil {
    private static SqlSessionFactory factory;  // 工厂对象（只创建一次）

    static {
        // static 代码块：类加载时自动执行，只执行一次
        factory = new SqlSessionFactoryBuilder()
                .build(Resources.getResourceAsReader("mybatis-config.xml"));
    }

    public static SqlSession getSqlSession() {
        return factory.openSession(true); // true = 自动提交事务
    }
}
```

> 💡 `SqlSession` 就是和数据库的一次"会话"，通过它可以执行 SQL。

### 6.5 mybatis-config.xml — MyBatis 全局配置

```xml
<configuration>
    <!-- 加载数据库配置 -->
    <properties resource="db.properties"/>

    <!-- 开启驼峰命名自动映射：emp_name → empName -->
    <settings>
        <setting name="mapUnderscoreToCamelCase" value="true"/>
    </settings>

    <!-- 数据库连接配置 -->
    <environments default="development">
        <environment id="development">
            <dataSource type="POOLED">  <!-- POOLED = 连接池 -->
                <property name="driver" value="${jdbc.driver}"/>
                <property name="url" value="${jdbc.url}"/>
                ...
            </dataSource>
        </environment>
    </environments>

    <!-- 注册所有 Mapper XML -->
    <mappers>
        <mapper resource="mapper/EmployeeMapper.xml"/>
        ...
    </mappers>
</configuration>
```

### 6.6 db.properties — 数据库连接信息

```properties

jdbc.username=root
jdbc.password=你的密码  <!-- 实际项目中应保密 -->
```

## 7. 第三层：业务逻辑层 Service（处理业务规则）

Service 层在 Mapper 之上，负责处理 **业务逻辑**。它调用 Mapper 的方法来完成具体的业务。

### 7.1 接口定义

```java
// EmployeeService.java — 接口（定义有哪些业务方法）
public interface EmployeeService {
    boolean add(Employee emp);
    boolean delete(Integer id);
    boolean update(Employee emp);
    Employee getById(Integer id);
    List<Employee> list();
}
```

### 7.2 实现类

```java
// EmployeeServiceImpl.java — 实现类（真正干活的代码）
public class EmployeeServiceImpl implements EmployeeService {

    // 拿到 Mapper，靠它操作数据库
    private final EmployeeMapper employeeMapper =
        MyBatisUtil.getSqlSession().getMapper(EmployeeMapper.class);

    @Override
    public boolean add(Employee emp) {
        return employeeMapper.addEmployee(emp);  // 直接委托给 Mapper
    }

    @Override
    public List<Employee> list() {
        return employeeMapper.listAll();  // 直接委托给 Mapper
    }
}
```

> 💡 员工和部门的 Service 比较简单，基本都是直接调用 Mapper（增删改查本来就是简单的操作）。

### 🔥 重点：考勤 Service — 有真正的业务逻辑！

```java
public class AttendanceServiceImpl implements AttendanceService {

    @Override
    public boolean checkIn(Integer empId) {
        // ① 先查今天是否已经打过卡
        Attendance today = attendanceMapper.getTodayAttendance(empId);
        if (today != null) return false;  // 已打卡，不能再打

        // ② 创建考勤记录
        Attendance att = new Attendance();
        att.setEmpId(empId);
        att.setCheckDate(new java.sql.Date(System.currentTimeMillis()));  // 今天日期
        att.setCheckIn(new Time(System.currentTimeMillis()));              // 当前时间

        // ③ 判断是否迟到：早于 9:00 = 正常，晚于 9:00 = 迟到
        Time now = new Time(System.currentTimeMillis());
        Time standard = Time.valueOf("09:00:00");
        att.setStatus(now.after(standard) ? "迟到" : "正常");

        // ④ 写入数据库
        return attendanceMapper.checkIn(att);
    }
}
```

> 💡 **这才是真正的业务逻辑！** 不是简单的增删改查，而是要判断"今天有没有打过卡"、"有没有迟到"。这些规则写在 Service 层，不写在 Servlet 层。

### 请假 Service

```java
public class LeaveServiceImpl implements LeaveService {

    @Override
    public boolean approve(Integer id) {
        return leaveMapper.updateStatus(id, "已通过");   // 审批通过
    }

    @Override
    public boolean reject(Integer id) {
        return leaveMapper.updateStatus(id, "已拒绝");   // 审批拒绝
    }
}
```

### 用户 Service — 注册逻辑

```java
public class UserServiceImpl implements UserService {

    @Override
    public boolean register(User user) {
        int rows = userMapper.insertUser(user);  // 插入数据库
        return rows > 0;  // 受影响行数 > 0 表示成功
    }

    @Override
    public User login(String username, String password, String role) {
        return userMapper.loginByRole(username, password, role);
        // 查到了返回 User 对象，查不到返回 null
    }
}
```

## 8. 第四层：控制器层 Servlet（接收网页请求）

Servlet 是 Java Web 的"门面"，负责接收浏览器发来的 HTTP 请求并返回响应。

### 8.1 什么是 Servlet？

```
用户在浏览器点按钮 → 发送 HTTP 请求到服务器
                         ↓
                    Servlet 收到请求
                         ↓
                 调用 Service 处理
                         ↓
                返回结果给浏览器（JSP 页面或重定向）
```

### 8.2 LoginServlet — 登录逻辑（最经典的 Servlet 例子）

```java
@WebServlet("/login")   // ← 注解：访问 /login 时触发这个 Servlet
public class LoginServlet extends HttpServlet {

    private final UserService userService = new UserServiceImpl();

    @Override
    protected void doPost(HttpServletRequest request,   // 请求对象（浏览器发来的）
                          HttpServletResponse response)  // 响应对象（返回给浏览器）
                          throws ServletException, IOException {

        // ========== 第一步：接收参数 ==========
        String username = request.getParameter("username");  // 获取表单中的用户名
        String password = request.getParameter("password");  // 获取表单中的密码
        String role = request.getParameter("role");          // 获取表单中的角色

        // ========== 第二步：调用业务层 ==========
        User loginUser = userService.login(username, password, role);

        // ========== 第三步：根据结果决定跳转 ==========
        if (loginUser == null) {
            // 登录失败 → 返回登录页，显示错误信息
            request.setAttribute("msg", "账号或密码错误！");
            request.getRequestDispatcher("/login.jsp").forward(request, response);
        } else {
            // 登录成功 → 把用户信息存入 Session
            HttpSession session = request.getSession();
            session.setAttribute("loginUser", loginUser);

            // 按角色跳转
            if ("admin".equals(loginUser.getRole())) {
                response.sendRedirect("/admin/index.jsp");     // 管理员后台
            } else {
                response.sendRedirect("/employee/index.jsp");  // 员工首页
            }
        }
    }
}
```

> 💡 **两个关键概念**：
>
> **forward（转发）**：服务器内部跳转，浏览器地址栏不变，可以携带 request 数据。
>
> **redirect（重定向）**：让浏览器重新请求新地址，地址栏会变，不能携带 request 数据。
>
> **Session（会话）**：服务器内存中的一块空间，存登录用户信息，一次登录期间有效。

### 8.3 EmpServlet — 员工管理（GET/POST 分离模式）

```java
public class EmpServlet extends HttpServlet {
    private EmployeeService employeeService = new EmployeeServiceImpl();

    // doGet 处理查询和删除
    @Override
    protected void doGet(HttpServletRequest request, HttpServletResponse response) {
        String action = request.getParameter("action");  // 通过 action 参数区分操作

        if ("list".equals(action)) {
            // 查询所有员工
            List<Employee> list = employeeService.list();
            request.setAttribute("empList", list);
            request.getRequestDispatcher("/admin/emp/list.jsp").forward(request, response);
        }
        else if ("delete".equals(action)) {
            // 删除员工
            int id = Integer.parseInt(request.getParameter("id"));
            employeeService.delete(id);
            response.sendRedirect("/emp?action=list");  // 删完跳回列表
        }
    }

    // doPost 处理新增
    @Override
    protected void doPost(HttpServletRequest request, HttpServletResponse response) {
        // 从表单获取数据，封装成 Employee 对象
        Employee emp = new Employee();
        emp.setEmpName(request.getParameter("empName"));
        emp.setGender(request.getParameter("gender"));
        emp.setDeptId(Integer.parseInt(request.getParameter("deptId")));

        employeeService.add(emp);
        response.sendRedirect("/emp?action=list");
    }
}
```

### 8.4 AttendanceServlet — 考勤打卡

```java
public class AttendanceServlet extends HttpServlet {

    @Override
    protected void doPost(HttpServletRequest request, HttpServletResponse response) {
        // ① 先检查是否登录
        HttpSession session = request.getSession();

            request.setAttribute("msg", msg);
            request.getRequestDispatcher("/employee/checkin.jsp").forward(request, response);

        } else if ("checkOut".equals(action)) {
            // 下班签退
            boolean result = attendanceService.checkOut(empId);
            String msg = result ? "下班签退成功！" : "未查询到今日上班打卡记录！";
            request.setAttribute("msg", msg);
            request.getRequestDispatcher("/employee/checkin.jsp").forward(request, response);
        }
    }
}
```

### 9.1 CharacterEncodingFilter — 统一设置编码

```java
public class CharacterEncodingFilter implements Filter {

    @Override
    public void doFilter(ServletRequest request, ServletResponse response, FilterChain chain)
            throws IOException, ServletException {
        request.setCharacterEncoding("UTF-8");   // 请求用 UTF-8
        response.setCharacterEncoding("UTF-8");  // 响应用 UTF-8
        response.setContentType("text/html;charset=UTF-8");
        chain.doFilter(request, response);  // 放行，继续交给下一个过滤器或 Servlet
    }
}
```

> 💡 **Filter 是什么？** 它就像一扇"安检门"，所有请求都要先经过它。这里用来统一设置中文编码，避免中文乱码。

## 10. 一个完整的请求是怎么走完的？

以 **"管理员查看所有员工列表"** 为例，追踪一次完整请求：

```
第 1 步：管理员在浏览器点击"员工管理"
         ↓
第 2 步：浏览器发送 GET 请求：http://localhost:8080/emp?action=list
         ↓
第 3 步：CharacterEncodingFilter 设置 UTF-8 编码
         ↓
第 4 步：EmpServlet.doGet() 收到请求
         ↓   读取 action=list
         ↓   调用 employeeService.list()
         ↓
第 5 步：EmployeeServiceImpl.list()
         ↓   调用 employeeMapper.listAll()
         ↓
第 6 步：MyBatis 找到 EmployeeMapper.xml 中的：
         <select id="listAll" resultType="entity.Employee">
             SELECT * FROM employee
         </select>
         ↓   执行 SQL，从数据库查出所有员工
         ↓   把每一行数据自动封装成 Employee 对象
         ↓   返回 List<Employee>
         ↓
第 7 步：回到 EmpServlet
         request.setAttribute("empList", list);         // 把数据放进 request
         request.getRequestDispatcher("/admin/emp/list.jsp").forward(...);  // 跳转页面
         ↓
第 8 步：list.jsp 用 JSTL 标签遍历 empList，生成 HTML 表格
         ↓
第 9 步：浏览器收到完整的 HTML 页面，渲染显示
```

**流程图：**

```
浏览器 → Filter → Servlet → Service → Mapper(MyBatis) → MySQL
                                                           ↓
浏览器 ← JSP(HTML) ← Servlet ← Service ← 结果(Entity列表) ←┘
```

### 11.1 user 表（登录用户）

```sql
CREATE TABLE `user` (
```

## 12. 总结：各层职责速查表

| 层 | 文件夹 | 职责 | 一句话 |
|----|--------|------|--------|
| **Entity** | `entity/` | 数据容器 | "和数据库表长得一模一样" |
| **Mapper** | `mapper/` + `resources/mapper/` | 执行 SQL | "Java 方法 ↔ SQL 语句" |
| **Service** | `service/` + `service/impl/` | 业务逻辑 | "判断迟到、审批请假" |
| **Servlet** | `servlet/` | 接收请求 | "服务员，接待浏览器请求" |
| **Filter** | `filter/` | 拦截请求 | "安检门，统一设编码" |
| **JSP** | `webapp/` | 显示页面 | "动态生成 HTML" |

3. **对比着看**：`EmployeeMapper.java` + `EmployeeMapper.xml` + `EmployeeServiceImpl.java` + `EmpServlet.java`，看一个功能怎么穿过四层。
4. **跑起来试试**：启动项目，在浏览器操作，同时在代码中打断点调试，感受数据流动。
